学习自这篇

问题

给一个固定的长度为 $2^N$ 的数组 $A$ 让你求 $F[x]$, $F[x]$ 定义如下：

朴素做法

对每个 $x$ 暴力枚举子集，有二项式定理可以证明这种做法是 $O(3^N)$ 的。

SOS ( Sum of Subsets ) DP

这其实是一种小技巧，类似滚动数组，虽然不学你也许能想出来，但还是学了方便很多。

在一开始我学 dp 的时候，我被教导说 dp 实际上是帮我们少做了很多重复性的操作，这种做法也是如此。在朴素做法中，我们在暴力枚举子集的时候做了很多重复的操作，而在这种做法中我们要减少这种重复操作的出现。

作者这里提供的转移方式是这样的：用 $dp[x][i]$ 表示从右往左扫了 $i+1$ 位的情况（就是后面 $i$ 位中 $1$ 转变为 $0$ 或者 $1$ 保持不变同统计了），那么 $dp[x][i]=dp[x][i-1]+dp[xXOR2^i][i-1]$ 具体可以参考原文中的图，十分清晰。

时间复杂度：$O(N2^N)$

F

有趣的一道数位 dp。

根据题意，我们要计算同类项个数，一个比较普遍的想法，我们对每一个同类项定义一个代表元素，让这个代表元素的字典序是这个同类项集合中最小的。

既然要最小就要满足一个要求，即这个元素不存在一串连续数字 $(x,d_1,d_2…d_k,y)$ 满足 $x>y$ 且存在变换 $(x,d1),(d1,d2)…(d_k,y)$ 。观察到事实上若我们已经知道后 $k+1$ 个数，我们只需考虑那些能移到最前面（$d_1$的位置）与 $x$ 的关系即可，这显然可以用二进制表达（tips: 外国人喜欢叫 mask）

这有点像数位 dp 尝试使用数位 dp 解决该问题。

让我们从右往左计算，令 $dp[x][mask]$ 表示我们从右往左计算到第 $x$ 位，且 $i\in mask $ 可以移到最左边。那么我们排除掉所有存在 $i<x，i\in mask$ 的 $x$ 即可，然后再算出新的 $mask$ 转移。

若暴力进行以上转移时间复杂度会爆掉 （ $O(n10^2\cdot2^{10})$ ）

所以我们要预处理 $mask$ 会转移到哪些 $x$ 上，且新的 $mask$ 是多少，这样时间复杂度就减到了 $O(10^2\cdot 2^{10}+n10\cdot2^{10})$

注意使用滚动数组来减少空间复杂度，还有就是如果使用了滚动数组你就要 memset ，你用了 memset 就会导致 long long 和 int 的时间差挺大的，于是要用 int

UVA1614 Hell on the Markets

发表于 2022-05-04 分类于 oi ， 贪心 阅读次数： Valine：

题目链接

思路

题目等价于求能否用 $a$ 中的部分数组成总和的一半。

观察到题目中有句 $a[i]\le i$ ，以此为突破口，发现貌似所有小于 $sum(a)$ 的数都可以用 $a$ 中的某些数组合出来，下证明：

  • 当 $n = 1$ 时，$1$ 显然能被 $1$ 组合出来

  • 若对于 $n$ 前 $n-1$ 个数的和为 $S_{n-1}$ 且所有小于 $S_{n-1}$ 的数都能被前 $n-1$ 个数表达出来，那么 $a[i]+1 \sim a[i]+S_{n-1}$ 显然都能被表示出来。

    • 由于 $a[i]<=S+1$ 故 $[a[i]+1,a[i]+S_{n-1}]\cup[1,S_{n-1}]=[1,S_{n}]$

    故所有小于 $S_{n}$ 的数都可以用前 $n$ 个数表达出来

结合上两条有：所有小于 $sum(a)$ 的数都可以用这 $n$ 个数表达出来。

所以我们只要排除 $sum(a)$ 为奇数的情况即可，具体搭配方法可以参考证明中的方法，尽量取大的即可。

时间复杂度

$O(nlogn)$

UVA1608 Non-boring sequences

发表于 2022-04-30 分类于 oi ， misc 阅读次数： Valine：

题目链接

思路

这题做法比较奇特，只能归为 misc

主题思想比较简单，就是对于一个子序列 $l \sim r$ 只要找到一个只出现一次的 $x$ 然后考虑子序列 $l\sim x-1$ 和 $x \sim r$ 即可

但奇特的点在于遍历方式，通过预处理可以让遍历验证 $x$ 的时间复杂度降为 $O( 1)$，预处理比较常规 $O(nlogn)$ 的离散化 + 找到左右最近的相同元素位置

如果我们从左往右朴素地遍历，那么 $T(n)=T(k)+T(n-k)+O(k)$ 最坏情况下为 $T(n)=O(n)+T(n-1)=O(n^2)$

于是考虑从两边向中间遍历，这样最左最右的情况就不是最劣，最劣时间复杂度会降为 $T(n)=2T(n/2)+O(n)=O(nlogn)$

从感性的角度讲就是不能 T , O 两个同时最差（虽然非常不严谨）

时间复杂度

$O(nlogn)$

UVA1471 - Defense Lines

发表于 2022-04-29 分类于 oi ， 贪心 阅读次数： Valine：

题目链接

思路

令 $r[i]$ 表示 $i$ 结尾最长严格上升序列，$l[i]$ 表示 $i$ 开头最长严格上升序列，原序列为 $a$

这两个序列都可以用 $O(n)$ 求得

只要能对于 $i$ 用较快时间找到相对应的 $r[i]$ 就可以解决这道题

观察到原题明显具有单调性的味道，故想到维护一个具有单调性的 $r[i]$ 序列

对于 $x$ 与 $i,j<x$ 先假设 $a[i],a[j]<a[x]$ 那么若 $a[i]<a[j]$ 且 $l[j]<=r[j]$ 那么 $j$ 就没有存在必要了，因为无论如何 $i$ 更优

根据上面这个性质可以想到维护一个集合 $s$ 其中，若 $a[i]<a[j]$ 则 $r[i]<r[j]$ 由于 c++ 中 set 可以用 lower_bound() 故可以通过 $O(nlogn)$ 解决这道题。

该题对指针理解要求较高，并有一些细节需要注意，我的程式属于写麻烦了，大家可以自行优化程序结构。

时间复杂度

$O(nlogn)$
